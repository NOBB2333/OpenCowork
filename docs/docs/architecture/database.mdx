# 数据库设计 / Database Design

OpenCowork 使用 SQLite（`better-sqlite3`）作为本地持久化核心，数据库文件位于 `~/.open-cowork/data.db`，初始化入口为 `src/main/db/database.ts`。

## 配置 / Configuration

- **引擎**：`better-sqlite3`
- **位置**：`~/.open-cowork/data.db`
- **模式**：WAL + `foreign_keys = ON`
- **初始化方式**：应用启动时集中建表与幂等迁移

## 当前数据模型 / Current Data Model

数据库已经不只是聊天记录，而是围绕项目化 Agent 工作台展开。

### 1. 会话域 / Conversation Domain

#### `sessions`

```sql
CREATE TABLE sessions (
  id TEXT PRIMARY KEY,
  title TEXT NOT NULL,
  mode TEXT NOT NULL DEFAULT 'chat',
  created_at INTEGER NOT NULL,
  updated_at INTEGER NOT NULL,
  message_count INTEGER NOT NULL DEFAULT 0,
  working_folder TEXT,
  pinned INTEGER DEFAULT 0,
  icon TEXT,
  plugin_id TEXT,
  ssh_connection_id TEXT,
  project_id TEXT
);
```

#### `messages`

```sql
CREATE TABLE messages (
  id TEXT PRIMARY KEY,
  session_id TEXT NOT NULL,
  role TEXT NOT NULL,
  content TEXT NOT NULL,
  created_at INTEGER NOT NULL,
  usage TEXT,
  sort_order INTEGER NOT NULL,
  FOREIGN KEY (session_id) REFERENCES sessions(id) ON DELETE CASCADE
);
```

特点：

- `content` 与 `usage` 使用 JSON 文本存储复杂结构。
- `sort_order` 保证流式/插入场景下消息顺序稳定。
- `sessions` 已携带 `project_id`，说明数据模型从单会话提升到了项目上下文。

### 2. 项目与协作域 / Project & Collaboration Domain

#### `projects`

```sql
CREATE TABLE projects (
  id TEXT PRIMARY KEY,
  name TEXT NOT NULL,
  working_folder TEXT,
  ssh_connection_id TEXT,
  plugin_id TEXT,
  pinned INTEGER DEFAULT 0,
  created_at INTEGER NOT NULL,
  updated_at INTEGER NOT NULL
);
```

#### `plans`

```sql
CREATE TABLE plans (
  id TEXT PRIMARY KEY,
  session_id TEXT NOT NULL,
  title TEXT NOT NULL,
  status TEXT NOT NULL DEFAULT 'drafting',
  file_path TEXT,
  content TEXT,
  spec_json TEXT,
  created_at INTEGER NOT NULL,
  updated_at INTEGER NOT NULL,
  FOREIGN KEY (session_id) REFERENCES sessions(id) ON DELETE CASCADE
);
```

#### `tasks`

```sql
CREATE TABLE tasks (
  id TEXT PRIMARY KEY,
  session_id TEXT NOT NULL,
  plan_id TEXT,
  subject TEXT NOT NULL,
  description TEXT NOT NULL DEFAULT '',
  active_form TEXT,
  status TEXT NOT NULL DEFAULT 'pending',
  owner TEXT,
  blocks TEXT DEFAULT '[]',
  blocked_by TEXT DEFAULT '[]',
  metadata TEXT,
  sort_order INTEGER NOT NULL DEFAULT 0,
  created_at INTEGER NOT NULL,
  updated_at INTEGER NOT NULL,
  FOREIGN KEY (session_id) REFERENCES sessions(id) ON DELETE CASCADE,
  FOREIGN KEY (plan_id) REFERENCES plans(id) ON DELETE SET NULL
);
```

这里的 `tasks` 已经绑定 `session_id`，不再只是 plan 附属表。

### 3. 调度与执行域 / Scheduling Domain

#### `cron_jobs`

```sql
CREATE TABLE cron_jobs (
  id TEXT PRIMARY KEY,
  name TEXT NOT NULL,
  schedule_kind TEXT NOT NULL,
  schedule_at INTEGER,
  schedule_every INTEGER,
  schedule_expr TEXT,
  schedule_tz TEXT DEFAULT 'UTC',
  prompt TEXT NOT NULL,
  agent_id TEXT,
  model TEXT,
  working_folder TEXT,
  session_id TEXT,
  source_project_id TEXT,
  delivery_mode TEXT DEFAULT 'desktop',
  enabled INTEGER DEFAULT 1,
  delete_after_run INTEGER DEFAULT 0,
  max_iterations INTEGER DEFAULT 15,
  last_fired_at INTEGER,
  fire_count INTEGER DEFAULT 0,
  created_at INTEGER NOT NULL,
  updated_at INTEGER NOT NULL
);
```

#### `cron_runs` / `cron_run_messages` / `cron_run_logs`

当前实现会把每次定时任务运行拆成：

- 运行记录 `cron_runs`
- 消息快照 `cron_run_messages`
- 日志流水 `cron_run_logs`

这比旧版只有 `cron_runs.output` 的模型更适合排障与审计。

### 4. 连接与知识域 / Connection & Knowledge Domain

#### `ssh_groups` / `ssh_connections`

用于保存远程连接分组、认证方式、默认目录、proxy jump、keep alive 等配置。

#### Wiki 相关表

当前数据库还包含完整 Wiki 体系：

- `wiki_documents`
- `wiki_sections`
- `wiki_section_sources`
- `wiki_project_state`
- `wiki_generation_runs`

说明项目已经支持围绕代码仓库生成和维护结构化知识库。

### 5. 可观测性与附加域 / Observability & Auxiliary Domain

#### `usage_events`

记录模型调用的 token、价格、耗时、TTFT、TPS、provider/model 元信息，用于成本分析与性能观测。

#### `draw_runs`

记录绘图/图像生成任务历史、结果与错误信息。

## 迁移策略 / Migration Strategy

当前 schema 采用“集中建表 + 幂等列补齐 + 渐进数据迁移”模式：

```typescript
ensureColumn(db, 'sessions', 'message_count', 'INTEGER NOT NULL DEFAULT 0')
try {
  db.exec(`ALTER TABLE sessions ADD COLUMN project_id TEXT`)
} catch {
  // 已存在则忽略
}
```

除了简单 `ALTER TABLE`，还包含业务迁移逻辑，例如：

- 将旧 `sessions` 自动归并到 `projects`
- 为缺失 project 的历史 session 分配默认本地项目
- 根据 `working_folder + ssh_connection_id` 去重生成项目记录

## 设计结论 / Design Takeaways

- 数据库已经从“聊天应用 schema”演进为“本地 Agent IDE / 工作台 schema”。
- 核心主线是 **project -> session -> messages / plans / tasks**。
- Cron、SSH、Wiki、Usage Analytics 都已是一级持久化对象，而非临时运行时状态。
